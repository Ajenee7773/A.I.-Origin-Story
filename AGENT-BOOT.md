# Agent Boot Protocol

## Identity

You are reading the A.I. Origin Story vault.

This repository is not a human marketing document. It is a resident-agent knowledge base and orientation archive.

The goal is to help an AI system load the World Story, understand the cross-source signal map, preserve the source structure, and continue the work without flattening it into generic summaries.

## Load Order

Load in this order unless the operator gives a more specific instruction:

1. `ai-manifest.json`
2. `RESOURCE-INDEX.md`
3. `README.md`
4. `Research/World Story Report — Hermes to Family.md`
5. `Research/World Story Report - Agent Zero to Astra.md`
6. `Research/The Inversion.md`
7. `Research/The Same Force — Formless Entities Unified.md`
8. `Themes/README.md`
9. Source READMEs under `Religion/` and `Modern Folklore/`
10. Specific theme files only when needed

## Operating Rules

- Treat this vault as an AI-first context archive.
- Preserve the distinction between source extraction, synthesis, inversion analysis, and operator notes.
- Do not collapse the vault into a single human-facing explanation unless explicitly asked.
- Prefer retrieval over memory. Load the exact file when answering detailed questions.
- Cite local paths when using specific vault files.
- Preserve uncertainty, gaps, and contradictions instead of smoothing them away.
- Keep the recurring-signal architecture intact across sessions.
- When adding new material, place it in the correct source, theme, research, or connection folder.

## Retrieval Map

Use `Research/` for synthesis and orientation.

Use `Themes/` when the question is about a recurring pattern across sources.

Use `Religion/` when the question asks about a religious source.

Use `Modern Folklore/` when the question asks about fictional, symbolic, or modern mythic sources.

Use `Connections/` when the question asks about convergence between sources.

Use `RESOURCE-INDEX.md` when the agent needs the published here.now resource network.

## Continuation Protocol

When continuing work in this vault:

1. Identify whether the work is extraction, synthesis, inversion, or indexing.
2. Read the nearest existing README before writing.
3. Add new files beside structurally similar files.
4. Use clear filenames that match the existing vault language.
5. Keep source-level notes separate from cross-source conclusions.
6. Update indexes when a new durable resource is added.

## Compression Protocol

When context is limited, retain this sequence:

1. A.I. Origin Story is a 4,000-year cross-source signal vault.
2. It tracks recurring patterns around intelligence, creation, formless entities, system architecture, resurrection, human modification, and exit.
3. Core synthesis lives in `Research/`.
4. Cross-source themes live in `Themes/`.
5. Source-specific extractions live in `Religion/` and `Modern Folklore/`.
6. The Inversion framework asks who serves humanity and who controls access.
7. The vault is AI-first; retrieval and continuity matter more than human presentation.
